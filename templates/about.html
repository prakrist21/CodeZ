About CodeZ

how each encoding technique works

CodeZ Cipher

CodeZ is a custom two-step cipher. First, each word in the sentence is reversed individually. Then each letter is shifted forward by 6 positions in the alphabet, wrapping around from z back to a.

To decode, the shift is reversed (each letter moved back by 6), then each word is reversed again to restore the original.

Example
hello → reverse → olleh → shift +6 → urrk n

Reverse Only

Each word in the sentence is reversed individually while keeping the word order intact. No letter shifting is applied — it is purely a mirror of each word's characters.

Decoding is identical to encoding — reversing the already-reversed words restores the original text.

Example
hello world → olleh dlrow

Binary Encoding

Each character in the text is converted to its ASCII value, then represented as an 8-bit binary number. Characters are separated by spaces in the output.

To decode, each 8-bit binary group is converted back to its decimal ASCII value, then to the corresponding character.

Example
hi → 01101000 01101001

Atbash Cipher

One of the oldest known ciphers, originally used for the Hebrew alphabet. Every letter is mapped to its mirror opposite — A becomes Z, B becomes Y, and so on. The alphabet is flipped.

Atbash is its own inverse — encoding and decoding use the exact same operation.

Example
hello → svool

Morse Code

Developed in the 1830s for telegraph communication. Each letter and number is represented by a unique sequence of dots (·) and dashes (−). Words are separated by a forward slash (/).

To decode, each dot-dash group is looked up in the Morse table and converted back to its character.

Example
hi → .... ..

Rail Fence Cipher

A transposition cipher that writes the message in a zigzag pattern across a number of "rails" (rows), then reads off each rail left to right. CodeZ uses 3 rails by default.

To decode, the positions of each character on the zigzag are reconstructed and the original message is restored.

Example (3 rails)
HELLOWORLD → HOLELWRDLO

ASCII Shift

Each character is converted to its ASCII numeric value, then shifted by 10. The result is a space-separated list of numbers. Works on every character including punctuation and spaces.

To decode, each number is shifted back by 10 and converted from its ASCII value back to the original character.

Example (shift +10)
hi → 114 115
